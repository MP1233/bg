Abstract

human cortex is 6 layered structure with primary and interneuron types studied. many of such types of neurons are already modelled in hoc, neuroML etc datasets. also there are many microcircuits models which describe how neocortex layers neurons are connected with each other. To test hypothesis how logic tasks are performed via such network, research on how these microcircuits function and simulation of different tasks is done via modelling. Number of models as well as software that perform such simulations exist (NEURON, NetPyne)
one of examples of tasks that human brain can perform - summing or substracting 2 values. although it is clear that such calculations are more complex and widespread across different areas of the brain, the purpose of this work is to implement a purely hypothetical microcircuit for simple calculations using already existing models of neocortex neurons to see how such microcircuit will appear in 3D. also to adjust such microcircuit to known charateristics of 6 layered structure, such as synapses and cell types density in the layers
although none of the microcircuits is supposed to work as a math calculator, it is quite possible that it can use some of similar algorythms for some of its calculations. especially this model performs calculations in the context of interactive input of values and operators which will involve communications between neighbouring areas. the purpose is to approximize how well can such type of algorythm fit the cortex 6 layered structure morphology
also this work can be used as practical experience with different neuroML, hoc, py type simulators (NEURON & NetPyne). most attention is done to adjusting synaptic mechanisms and HH parameters task

In the first part "SUMM BLOCK" is provided with architecture that simply activates "summ RESULT neuron" when two value neurons "COMPONENT1 & 2" start firing. Also optimal filter mechanism is selected at this step. In the second part "CONTEXT BLOCK" is added to the scheme. This context block processes input from user and sends it to "SUMM BLOCK". 2 blocks are connected via BG and thalamus part. Their addition allows to process more complex formulas such as A + B - C + D =. User input is similar to standart calculator interface. Third part replaces logical scheme with "ball & stick neurons" in netpyne simulation software and further with real neocortex neuron models from EPFL data. Classical neocortex layer networks are used to transform logical schemes onto neocortex 3D volume. Different data input will be tested from pulse to tonic input. Fourth chapter compares synapse and cell locations densities with statistics from neocortex

Further research is planned for a number of different logical computing tasks to compare which of them best fit cortex layers framework. Some off-topic research is possible, for example calculate energy consumption of synapses and try to minimize it in the model (or fit it to BOLD signal fMRI statistics). Or even compile computer code into seqeunces of neurons microcircuits data collected from different sources - total_info.htm

Now we have microcircuits that are connected with other microcircuits by forward and feecback inter-microcircuits connections (also see Fig). 1st microcircuit summs or minus components. 2nd microcircuit processes input signals. 3rd transaction microcircuit waits while information input is complete and the time is right to commit the transaction Microcircuits in addition to direct connections also connected via the relay mechanism, which better suits thalamus.
We have modules that process and store data [context block, summ block]. And we have modules that process external commands and coordinate data modules

This is done via infinite cortico-thalamo-cortical loop. SUMM BLOCK: summ components are kept in Layer2/3 memory via recurrent loop. current summ result is kept in Layer5 via recurrent loop and avaits for command from thalamus. CONTEXT BLOCK: keeps last user input via Layer4 recurrent loop. When the user adds new operation and component to the input formula - thalamus opens the block and inputs to SUMM BLOCK previous result from the SUMM BLOCK and user input from CONTEXT BLOCK. SUMM BLOCK recalculates its result and waits for new Thalamus unblock
This SUMM BLOCK+INPUT > SUMM BLOCK loop continues until user presses "=" button

Initial model for "SUMM" block will be

Functional pipeline:

Layer 4 receives input from ThalCore and amplifies it [none for our model]
Layer 3 context input from higher order cortex [0 1 2 3 + -] ThalMatrix [start command] and layer 4 [none]
Layer 3 function extrapolate [all possible combinations 0+0 ... 3-3] and winner takes all inh filter [L2/3P = 0+0 0+1 .... ; L2/3Basket = common filter for winner]
Layer 3 output layer 5
Layer 5 input from layer 3, context from higher area [none] context from ThalMatrix [green light to transfer summ result]
Layer 5 function transform all possible combinations 0+0 ... 3-3 to -3 ... 6. Wait for green light to proceed output [-3 ... 6] //Burst firing via activation of Ca 2+ action potential in the apical tufts of the pyramidal cells
Layer 5 output a) higher cortex [-3 .. 6] (CONTEXT BLOCK) b) to Layer x6x 5B [calculation finished] c) to spine [final result action][none: L5P+L5BVertical > L2/3P = what feedback? context? only if signal was noisy]
*Problem for higher cortex output: L5 projects to L5syn, but we need L3syn. But such link can only be L5 > L5B > Thal > L1syn > L2/3 {L2/3 of S1 > L2/3 of S2 or directly to L2/3 of S2}. Or some axon long range CC projections (but they are contralateral). Or subset of L6B neurons send also axons to layer 1 (Mitchell and Cauller, 2001) ; Or L5B > MatrixThal > ThalCore > L4 > L2/3 (Theyel et al.,2010) suggesting a corticothalamocortical pathway via layer 5B from S1 cortex to higher-order sensory cortices such as the S2 cortex ; Axons from the primary whisker motor cortex (M1) arborized densely in L1, and more diffusely in L5 and L6 ;
Layer 5A recurrent excitation slender-tufted (L5A) [5A waits, 5B transmits signals when 5A+Thal actiated]
Layer 5B>6A or 5B output ThalMatrix [calculation finished]
Layer6 none [modulation of Thal ascending sensory impulse to L4] [none for our model]
Layer6 none [feedforward via Thal to other cortex areas] [none for our model]

*Also estimate proportion of inhibitory neurons. and test which scheme best applies to competitor wins scheme: 1 population AExc has its own population AInh which blocks all others populations Bexc Cexc Dexc ... vs 2 populations Aexc Bexc Cexc Dexc ... have common population IInh which blocks all Aexc Bexc Cexc Dexc.... ("competition through shared inhibition")
"competition through shared inhibition" model is preferred by several reasons 1) number of inhibitory interneurons should be at most in range 10-20%. this model will reduce number of interneurons 2) model is well implemented in many simulations 3) in-vivo we don't need extra presumption that precise neuron microcircuits resemble computer code, so we prefer to be as abstract as possible.
Also when we activate components such as 1 2 3 + - etc - should we activate all neuron populations that represent them, or should we activate only some of them with further activation of whole population by the Hopfield network. Adjusting population sizes both competitive and Hopfield models could be implemented for the same populations how working memory is implemented into loops - asynchronic tonic or oscilations? also what types of cells involved?

to control signal start several options are possible 1) signal is active + but blocked by inhibitory modulation - 2) signal is balanced between inh exc subpopulations and 0, starts when excitory signal from thalamus arrives
